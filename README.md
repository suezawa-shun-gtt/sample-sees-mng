# SEES Management System

サービス終了サイト（Service End Spot）管理システム

## 概要

東京都政が運用するサービスは多数あり、既に終了したサービスも少なくありません。しかし、別ページからの参照リンクがそのまま残っていることが多く、アクセスすると404エラーが表示されてしまいます。

本システムは、終了したサイトへのアクセスに対して適切な案内HTMLを表示し、数秒後に指定のページへリダイレクトする仕組みを一括管理するための管理サイトです。

## 主な機能

### 1. SEES情報管理
- **一覧表示**: 登録されているSEES情報の一覧を表示
- **追加**: 新規SEES情報の登録
- **編集**: 既存SEES情報の修正
- **削除**: 不要なSEES情報の削除

### 2. SEES情報の構成要素
各SEESは以下の情報で構成されます：
- **対象ドメイン**: 終了したサービスのドメイン
- **リダイレクト先URL**: 案内後に遷移させる先のURL
- **表示用HTML**: リダイレクト前に表示する案内HTML
  - 汎用的なデフォルトテンプレートを提供
  - SEES毎にカスタマイズ可能

### 3. Azure自動構築機能
SEES登録後、以下のAzureリソースを自動構築：
- **DNSゾーン**: 対象ドメインを管理
- **Static Web App**: 案内HTMLをホスティング
- 対象ドメインへのアクセスをStatic Web Appへルーティング

### 4. NSレコード管理
- 作成したDNSゾーンのNSレコードを表示
- NSレコードを元のDNSに手動登録し、DNSゾーンへ委任
- 各SEESのNSレコードをシステム内で管理

## システム構成

```
sample-sees-mng/
├── sees-console/          # 管理画面（Next.js）
│   ├── app/              # Next.js App Router
│   ├── prisma/           # データベーススキーマ
│   └── public/           # 静的ファイル
└── README.md             # このファイル
```

## 技術スタック

- **フロントエンド**: Next.js 15 (App Router)
- **データベース**: Prisma ORM
- **クラウド**: Azure (DNS Zone, Static Web App)
- **言語**: TypeScript
